# PUW terminy bot

Telegram-бот, который следит за системой бронирования Gdańsk PUW (Qmatic) и присылает
уведомление, как только появляются свободные термины.

## Установка

```bash
npm install
cp .env.example .env
```

Получи токен у [@BotFather](https://t.me/BotFather): `/newbot` → имя → username → токен.
Вставь его в `.env` в поле `BOT_TOKEN`.

## Запуск

```bash
npm start        # запуск (tsx, без сборки)
npm run dev      # с авто-перезапуском при правках
```

Затем напиши боту `/start` в Telegram — он запомнит твой чат и начнёт уведомлять.

## Команды бота

- `/start` — подписаться на уведомления
- `/stop` — отписаться
- `/status` — статус подписки и настройки
- `/check` — проверить свободные термины прямо сейчас

## Как работает

Бот раз в `POLL_INTERVAL_SEC` секунд (по умолчанию 60) дёргает Qmatic REST API:
сначала список свободных дат, затем конкретные часы для каждой даты. Уведомление шлётся
только при появлении **новых** дат, чтобы не спамить. При старте бот делает «нулевой»
замер — текущие открытые даты не считаются новыми.

## Настройка цели (`.env`)

`BRANCH_ID` / `SERVICE_ID` по умолчанию указывают на услугу «obywatelstwo» в гданьском
PUW. Чтобы следить за другой услугой/отделом — открой сайт бронирования, в DevTools
(Network) найди запрос `.../rest/schedule/branches/<branch>/dates;servicePublicId=<service>...`
и подставь свои ID.

## Заметки

- Окно бронирования в PUW открывается максимум на 2 недели вперёд; новые места
  выкладываются после полуночи и быстро разбираются — держи бота запущенным ночью/утром.
- Если API начнёт отдавать HTTP 4xx — возможно, нужна cookie-сессия. Дай знать, добавлю
  «прогрев» сессии (GET на базовую страницу перед опросом).
- Минимальный интервал опроса ограничен 30 c — будь вежлив к серверу.

### TLS

Сервер PUW отдаёт неполную TLS-цепочку (без промежуточного сертификата Certum), из-за
чего Node-ский `fetch` падал с `UNABLE_TO_VERIFY_LEAF_SIGNATURE`. Недостающий
промежуточный лежит в [certs/certum-dv-tls-g2-r39.pem](certs/certum-dv-tls-g2-r39.pem) и
подключается через `NODE_EXTRA_CA_CERTS` прямо в npm-скриптах — проверка TLS остаётся
включённой, ничего отключать не пришлось. Поэтому запускай именно через `npm start`
(не `node`/`tsx` напрямую, иначе переменная не подхватится).
